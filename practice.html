Разработка диаграммы компонентов

Диаграмма компонентов - статическая структурная диаграмма, показывает разбиение программной системы на структурные компоненты и связи (зависимости) между компонентами. В качестве физических компонентов могут выступать файлы, библиотеки, модули, исполняемые файлы, пакеты и т. п.

Рассмотрим структуру приложения под операционную систему iOS. Особенностью этой операционной системы является то, что любая программа представляет собой пакет, внутри которого содержатся все остальные файлы. Этот пакет имеет расширение .ipa. Каждый .IPA файл - это бинарный файл для ARM-архитектуры, который на деле является сжатой в формате ZIP папкой определённой структуры и содержащей исполняемый файл, файлы ресурсов, таких как видео, аудио и изображения и т. д. На рисунке 1 приведена структура приложения, или другими словами – диаграмма компонентов.



Рисунок 1 - Диаграмма компонентов проектируемой системы

Приведем подробное описание некоторых компонентов компонента.

  • beINSPORTS.app – это исполнительный файл, который содержит программный код приложения и главную точку входа.
  • ресурсы – это место, хранятся различные изображения, графические элементы приложения. Они используются в определенное время в приложении.
  • Info.plist – информационный список свойств. Это структурированный файл, который содержит информацию о конфигурациях приложения. Он является представлением иерархии объектов, которые могут храниться в файловой системе и могут быть восстановлены позже. Список свойств легко и компактно хранит небольшие объемы данных. Он состоит из конкретных типов данных.
  • Layout файлы – это файлы приложения, который содержит объекты пользовательского интерфейса, загружаемые в приложения в момент его запуска. Имеет xib или storyboard форматы данных.
  • Resources – это какие-либо ресурсы. Подразделяются на локализированные и не локализированные. Не локализированные ресурсы находятся в каталоге верхнего уровня, а локализированные - в специальных языковых подкаталогах пакета приложения. Ресурсы содержат в себе изображения, звуковые, файлы пользовательского интерфейса, текстовые файлы и другие данные, которые необходимы для приложения.
  • Frameworks – это пакеты, представляющие собой комплект (структурированные каталоги), которые содержат динамические разделяемые библиотеки вместе с соответствующими ресурсами.

Разработка структурной схемы системы

Структурная схема системы показывает, с какими уровнями и системами взаимодействует разрабатываемая система.

Структурная схема проектируемой системы представлена на рисунке 2. Рассмотрим, как же с мобильного аппарата передается информация на сервер.

Первым делом мобильный аппарат посылает радиосигналы на базовую станцию. Базовая станция – это комплекс радиопередающей аппаратуры (ретрансляторы, приёмопередатчики), осуществляющий связь с конечным абонентским устройством – сотовым телефоном.



Рисунок 2 - Структурная схема проектируемой системы

На базовой станции установлен сетевой шлюз. Сетевой шлюз – аппаратный маршрутизатор или программное обеспечение для сопряжения компьютерных сетей, использующих разные протоколы (например, локальной и глобальной).

Далее информация проходит через фаервол. Фаервол (firewall) – это преимущественно основной компонент необходимый для организации защиты периметра сети. Фаервол - это система либо группа систем, контролирующая доступ между двумя и более сетями. Функция фаервола заключается в том, чтобы разрешить либо запретить трафик, проходящий через него, основываясь на определенных правилах. Все фаерволы выполняют функцию анализа сетевого трафика и направляют его на основе набора правил, однако, методы, которые используют для этого фаерволы, могут различаться. Информация приходит на сервер, где обрабатывается и записывается в базу данных.

Так же определяется текущее положение пользователя смартфона при помощи GPS или ГЛОНАСС. GPS (англ. Global Positioning System – система глобального позиционирования) – спутниковая система навигации, обеспечивающая измерение расстояния, времени и определяющая местоположениe во всемирной системе координат WGS 84. Позволяет в любом месте Земли (исключая приполярные области), почти при любой погоде, а также в околоземном космическом пространстве определять местоположение и скорость объектов.

Глобальная навигационная спутниковая система (ГЛОНАСС) - советская и российская спутниковая система навигации, разработана по заказу Министерства обороны СССР. Одна из двух функционирующих на сегодня систем глобальной спутниковой навигации (китайская система спутниковой навигации Бэйдоу на данный момент функционирует как региональная).

ГЛОНАСС предназначена для оперативного навигационно-временного обеспечения неограниченного числа пользователей наземного, морского, воздушного и космического базирования. Для определения координат приёмник должен принимать сигнал как минимум четырёх спутников и вычислить расстояния до них. При использовании трёх спутников определение координат затруднено из-за ошибок, вызванных неточностью часов приёмника.

Обобщенный алгоритм функционирования системы

При проектировании была разработана функциональная модель программы. На основе этой модели были построены алгоритмы. Обобщенный алгоритм работы системы представлена на рисунке 3.

В начале работы программы пользователь попадает на главную страницу, затем он может открыть главное меню, где он сможет выбрать один из вариантов развития событий или взаимодействовать с главным экраном.

При выборе пункта ТВ расписание, пользователю будет предложено выбрать телеканал, затем можно выбрать другой день и нажать на доступные видео. Затем пользователь может начать смотреть видео на мобильной платформе или начать трансляцию через специальное устройство подключенное к ТВ.



Рисунок 3 - Схема обобщенного алгоритма работы системы

При выборе пункта Виды спорта, пользователю дается возможность выбрать нужный спорт, после чего для каждого из видов будут показаны свои подкатегории (Новости, Видео, Результаты, Статистика).

При выборе Новостей, пользователей видит список последних статей, а так же может открыть их более подробное описание.

При выборе Видео, пользователь может начать смотреть видео на телевизоре, с помощью устройства или смотреть на мобильном телефоне.

На странице результатов, пользователю будет показан список последних и будущих игр, в зависимости от выбранного вида спорта.
